## 前引
随着科技的飞速发展，无人机已广泛渗透至军事与民用各大领域，其市场潜力持续扩大。作为高科技、高精度的代表，无人机虽价值不菲，但其安全性，尤其是丢失风险，成为监管部门与用户共同关注的焦点。为此，创新性地研发了一套无人机丢失找寻系统，旨在为用户提供高效、便捷的解决方案。
#
## 项目简介
随着无人机在军事侦察、电力巡检、林区监测、测绘勘探、应急救援等场景中的广泛应用，其在复杂环境下的飞行安全问题日益突出。尤其在**山区、森林、峡谷、强磁干扰区、弱通信覆盖区**等环境中，无人机容易出现偏航、剧烈姿态异常、失联甚至坠落等情况，一旦发生丢失，传统依赖人工回收与飞控日志分析的方式往往效率低、定位误差大、搜寻周期长。
针对上述问题，本项目设计并实现了一套**复杂环境无人机丢失找寻系统**。系统采用**双 STM32 主控协同架构**，融合**多传感器采集、MAVLink 飞控数据解析、FreeRTOS 实时任务调度、NanoEdge AI 端侧异常识别**以及**4G 云端数据回传**等关键技术，实现以下目标：
- 对无人机姿态、海拔、位置信息、飞行状态进行实时采集与分析；
- 对异常飞行状态进行快速识别与告警；
- 在无人机发生失联或坠落时，保留最后有效位置并上传云平台；
- 为后续人工搜寻或系统辅助回收提供可靠的数据依据。
本系统面向复杂环境下无人机“**监测—诊断—告警—定位—找回**”全流程需求，具备较强的工程实用价值和扩展能力。
---
## 项目目标
本系统主要解决以下实际问题：
1. **复杂环境下无人机状态难以稳定感知**  
   传统单一传感器方案难以对突发异常进行准确判断。
2. **无人机失联后缺乏有效回溯与定位依据**  
   飞控日志依赖事后提取，实时性差，无法第一时间提供搜寻坐标。
3. **单 MCU 架构下任务负载过高**  
   高频传感器采样、飞控串口解析、AI 推理、4G 通信并发运行时容易造成阻塞和丢包。
4. **复杂环境中的通信链路不稳定**  
   飞控到地面站链路在遮挡环境中容易中断，导致搜寻窗口极短。
因此，本项目采用云边协同方案，在机载端完成关键状态感知与异常检测，在云端完成远程追踪与信息展示。

## 核心特性
### 1. 双 STM32 主控协同架构
系统使用两颗 STM32L4 系列 MCU 进行分工协作：
- **主控 1：STM32L431RCT6**
  - 负责传感器数据采集与融合
  - 负责异常检测与业务逻辑执行
  - 运行 FreeRTOS 多任务系统
  - 负责 4G 数据上传与云端交互
- **主控 2：STM32L4CCT6**
  - 负责与无人机飞控通信
  - 负责 MAVLink 1.0 协议解析
  - 提取飞控关键信息并转发至主控 1
通过双核分工，可有效降低单片机任务负载防止系统冗余，提升系统的实时性与稳定性。
### 2. 端侧 AI 异常状态识别
引入 **NanoEdge AI** 轻量化边缘智能算法，在 MCU 端直接运行异常识别模型，不依赖上位机或高性能处理器，能够对以下异常进行快速检测：
- 无人机剧烈震荡
- 姿态异常偏航
- 自由跌落/坠落趋势
- 飞行状态突变
该方案具备低功耗、低延迟、边缘自治能力强等特点，适合嵌入式机载设备部署。
### 3. 多源传感器融合感知
系统通过多种机载传感器实现状态感知：
- **MPU6050**：获取三轴加速度、三轴角速度，用于姿态与运动状态分析
- **MS5611**：获取气压与高度信息，用于飞行海拔判断
- **GPS / 飞控位置信息**：获取经纬度、航向、速度等关键导航参数
通过对不同传感器信息的联合分析，可提高异常识别与失联定位的可靠性。
### 4. MAVLink 飞控数据解析
主控 2 通过 UART 与无人机飞控 **Telem** 接口相连，基于 **MAVLink 1.0** 协议进行通信，实时解析以下信息：
- 无人机电池电量
- 飞行速度
- 飞行航向
- GPS 坐标
- 飞行模式与系统状态
该部分是系统获取飞控核心状态的重要数据来源。
### 5. 4G 云端回传与远程追踪
主控 1 通过 4G 无线透传模块将关键状态数据上传至云平台（如原子云 / 物联网平台），实现：
- 实时位置上传
- 异常状态上报
- 最后有效坐标锁定
- 云端可视化展示与远程访问
当无人机失联或坠落时，系统将尽可能保留事故前最后一组有效状态数据，辅助后续搜寻。
### 6. FreeRTOS 实时调度
系统基于 **FreeRTOS** 构建多任务框架，将采集、解析、判断、通信等功能解耦为独立任务，提升系统的并发处理能力与工程可维护性。
